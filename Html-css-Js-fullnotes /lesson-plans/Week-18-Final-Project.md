# WEEK 18 LESSON PLAN

## 1. Lesson Information

| Field | Value |
|-------|--------|
| **Week** | 18 |
| **Topic** | Final Project: Build, Showcase, and Reflection |
| **Duration** | 2–3 Hours |

---

## 2. Learning Objectives

By the end of this lesson, students will be able to:

- Deliver a multi-page site that meets the rubric and runs without console errors.
- Present the project briefly and answer reflection questions (spoken or written).
- Use the [Final-Project-Rubric.md](../notes/Week-6/Final-Project-Rubric.md) for self- and peer assessment.

---

## 3. Prior Knowledge

Students should have completed:

- Weeks 1–15 (HTML, CSS, JS including DOM, events, arrays, localStorage).
- Week 16 (project structure; optional Git).
- Optionally Week 17 (Fetch API).

---

## 4. Materials Needed

- Computer, code editor, browser
- [Final-Project-Rubric.md](../notes/Week-6/Final-Project-Rubric.md) — share again at start of session
- [Common-Mistakes-And-Debugging.md](../notes/Week-1/Common-Mistakes-And-Debugging.md) — **Session 15** for final debugging tips

---

## 5. Key Concepts

- **Bringing it together:** HTML (structure, semantics), CSS (layout, responsiveness), JS (interactivity), clean code, creativity.
- **Rubric:** HTML 20%, CSS 25%, JS 25%, code cleanliness 15%, creativity 15%.
- **Reflection:** What was hardest? What did you debug? What would you improve?

---

## 6. Lesson Procedure

### A. Final Checklist & Rubric (15–20 minutes)

- Share [Final-Project-Rubric.md](../notes/Week-6/Final-Project-Rubric.md) again.
- Quick debugging tips: [Common-Mistakes-And-Debugging.md](../notes/Week-1/Common-Mistakes-And-Debugging.md) — **Session 15**.
- Remind students of minimum requirements (see below).

### B. Build and Polish (60–90 minutes)

Students build or continue their final project. Instructor circulates to support. Ensure:

- At least: Home, About, Projects, Contact (or equivalent).
- Semantic HTML; consistent nav; external CSS; responsive.
- At least two interactive features (e.g. mobile menu, theme toggle, form validation, dynamic list, or fetch + list).
- Clean folders and a short README.

### C. Demos and Reflection (25–35 minutes)

- Short demos: 1–2 minutes per student (or pair).
- **Reflection** — each student answers (spoken or written):
  1. What was the hardest part?
  2. What’s one bug you fixed or thing you debugged?
  3. What would you improve or add if you had more time?

### D. Assessment

Assess with the rubric:

| Criterion | Weight |
|-----------|--------|
| HTML (structure, semantics) | 20% |
| CSS (layout, responsiveness) | 25% |
| JavaScript (interactivity) | 25% |
| Code cleanliness | 15% |
| Creativity | 15% |

**Milestone:** Frontend Project Developer.

---

## Final Project Requirements (Summary)

- **Pages:** At least Home, About, Projects, Contact (or equivalent). Semantic HTML; consistent nav; external CSS; responsive.
- **Interactivity:** At least two features (e.g. mobile menu, theme toggle, form validation, dynamic list, fetch + list).
- **Quality:** Clean folders; short README; no console errors.
- **Real-world:** “Portfolio-ready: host on GitHub Pages or Netlify and link from your resume or school profile.”

---

## Common Mistakes This Week

*Use [Common-Mistakes-And-Debugging.md](../notes/Week-1/Common-Mistakes-And-Debugging.md) — **Session 15** (5–10 min).*

- Leaving console errors; forgetting to test on different screen sizes.
- Not checking links and paths after last-minute changes.
