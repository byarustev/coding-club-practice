# WEEK 7 LESSON PLAN

## 1. Lesson Information

| Field | Value |
|-------|--------|
| **Week** | 7 |
| **Topic** | Responsive Design: Viewport, Units, Media Queries |
| **Duration** | 2–3 Hours |

---

## 2. Learning Objectives

By the end of this lesson, students will be able to:

- Make one page readable on narrow (mobile) and wider (desktop) screen.
- Use at least one breakpoint and rem for font sizes.
- Make the Week 6 card page responsive: small screens = one column; larger = 2–3 columns.

---

## 3. Prior Knowledge

Students should understand:

- Box model, classes, basic CSS.
- Card layout from Week 6.

---

## 4. Materials Needed

- Computer, code editor, internet access
- Week 6 card page

---

## 5. Key Concepts

- Why responsive design matters.
- Viewport meta tag.
- Relative units (% , rem) vs px.
- Media queries (e.g. 768px); mobile-first: base for small, then min-width for larger.

---

## 6. Lesson Procedure

### A. Common Mistakes Review (5–10 minutes)

Use [Common-Mistakes-And-Debugging.md](../notes/Week-1/Common-Mistakes-And-Debugging.md) — **Session 5** (responsive). Viewport meta, min-width vs max-width, rem for text.

### B. Introduction of Core Concept (40–60 minutes)

Instructor explains: viewport meta; px vs rem and %; one or two media queries (e.g. min-width: 768px); mobile-first approach; live coding: base styles then media query for wider layout.

### C. Guided Practice (30–45 minutes)

Students: ensure viewport meta; convert key font sizes to rem; add media query so cards stack on small screen and sit in 2–3 columns on large.

### D. Independent Practice (30–60 minutes)

**Main exercise:** Make the Week 6 card page responsive. Small screens: cards stack (one column, full width). Larger screens: 2–3 columns (e.g. % or flex). Use rem for font sizes.

**Real-world:** “Every modern site needs to work on phones and desktops.”

### E. Assessment

Page readable on narrow and wide viewport; at least one breakpoint; rem used for fonts; no console errors.

### F. Wrap-Up Discussion (10–15 minutes)

What was difficult? What did you debug? How is this used in real life?

### G. Homework (Light / Optional)

Try Flexbox Froggy or read “Flexbox basics” in Week 3 material.
