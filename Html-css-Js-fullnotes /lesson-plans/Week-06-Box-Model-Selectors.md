# WEEK 6 LESSON PLAN

## 1. Lesson Information

| Field | Value |
|-------|--------|
| **Week** | 6 |
| **Topic** | Box Model and Selectors (Classes & IDs) |
| **Duration** | 2–3 Hours |

---

## 2. Learning Objectives

By the end of this lesson, students will be able to:

- Add padding, margin, and border to sections or cards.
- Use one class for all cards and one ID for one special element.
- Build one HTML page with 2–3 “cards” using a single class; different margin/padding; simple hover effect.

---

## 3. Prior Knowledge

Students should understand:

- Linking CSS and basic properties (color, font-size, etc.).
- HTML structure and semantic tags.

---

## 4. Materials Needed

- Computer, code editor, internet access
- Previous project or new page for cards

---

## 5. Key Concepts

- **Box model:** content, padding, border, margin.
- **Classes** for repeated styles; **IDs** for a single element.
- Specificity in simple terms.
- box-sizing: border-box; minimal reset.

---

## 6. Lesson Procedure

### A. Common Mistakes Review (5–10 minutes)

Use [Common-Mistakes-And-Debugging.md](../notes/Week-1/Common-Mistakes-And-Debugging.md) — **Session 4** (box model). Specificity, one ID per page, box-sizing.

### B. Introduction of Core Concept (40–60 minutes)

Instructor explains: box model (content, padding, border, margin); class vs ID; selector syntax for .class and #id; box-sizing: border-box; live coding of one card with .card.

### C. Guided Practice (30–45 minutes)

Students: create 2–3 card elements with same class; add padding, border, background to .card; add margin between cards; add hover (border or box-shadow).

### D. Independent Practice (30–60 minutes)

**Main exercise:** One HTML page with 2–3 “cards” (product, service, or hobby). One class (e.g. .card) for padding, border, background; different margin/padding for card vs text inside; simple hover (border or shadow).

**Real-world:** “Same idea as product or feature cards on a website.”

### E. Assessment

Cards styled with class; padding/margin/border applied; no console errors.

### F. Wrap-Up Discussion (10–15 minutes)

What was difficult? What did you debug? How is this used in real life?

### G. Homework (Light / Optional)

Read about responsive design and viewport meta tag. Add viewport meta if not already there.
