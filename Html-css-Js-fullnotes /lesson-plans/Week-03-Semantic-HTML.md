# WEEK 3 LESSON PLAN

## 1. Lesson Information

| Field | Value |
|-------|--------|
| **Week** | 3 |
| **Topic** | Semantic HTML: Structure and Meaning |
| **Duration** | 2–3 Hours |

---

## 2. Learning Objectives

By the end of this lesson, students will be able to:

- Wrap content in the right semantic tags (`header`, `nav`, `main`, `section`, `article`, `footer`).
- Explain in one sentence why semantic HTML is used (accessibility, SEO).
- Add semantic structure to the Week 2 page: header (with h1), nav (anchor links to sections), main, at least two sections, footer.

---

## 3. Prior Knowledge

Students should understand:

- Essential HTML tags: headings, paragraphs, links, images, lists.
- Valid document structure.

---

## 4. Materials Needed

- Computer
- Code editor
- Internet access
- Week 2 “About me” page

---

## 5. Key Concepts

- **Semantic HTML:** `header`, `nav`, `main`, `section`, `article`, `footer`.
- When to use them instead of plain `div`.
- Why it helps accessibility and SEO.

---

## 6. Lesson Procedure

### A. Common Mistakes Review (5–10 minutes)

- Use [Common-Mistakes-And-Debugging.md](../notes/Week-1/Common-Mistakes-And-Debugging.md) — **Session 3**.
- Review link paths and structure (e.g. nav links to sections).

### B. Introduction of Core Concept (40–60 minutes)

Instructor explains:

- What “semantic” means: tags that describe meaning, not just layout.
- `header`: top of page (often with h1, logo).
- `nav`: navigation links.
- `main`: main content (one per page).
- `section` / `article`: logical blocks.
- `footer`: bottom of page.
- Live coding: wrap existing “About me” content in semantic tags.

### C. Guided Practice (30–45 minutes)

Students follow instructor to:

- Add `header` with `h1`.
- Add `nav` with anchor links to section IDs.
- Wrap content in `main` and at least two `section`s.
- Add `footer`.

### D. Independent Practice (30–60 minutes)

**Main Weekly Exercise:**

Take the Week 2 page and add semantic structure. Must include:

- `header` (with `h1`).
- `nav` (anchor links to sections).
- `main`.
- At least two `section`s.
- `footer`.
- Keep content the same; only structure changes.

**Real-world:** “Sites and screen readers use this structure to understand the page.”

### E. Assessment

Students must demonstrate:

- Correct semantic wrapping.
- Working anchor links in nav.
- No structural errors.

### F. Wrap-Up Discussion (10–15 minutes)

- What was difficult?
- Why do we use semantic HTML?
- How is this used in real life?

### G. Homework (Light / Optional)

- Read “What is CSS?” and “Linking a stylesheet” in Week 2 material.
- Add `<link rel="stylesheet" href="styles.css">` to HTML and create an empty `styles.css`.
